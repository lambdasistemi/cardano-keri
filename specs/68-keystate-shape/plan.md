## Canonical CBOR / byte-identity mechanism

One Haskell computation produces every canonical byte string. The generator writes
(a) the JSON golden/negative vectors and (b) an Aiken fixtures module of the same
hex. The Haskell tests assert `haskell_encoder(value) == json_bytes`; the Aiken
tests assert `cbor.serialise(aiken_value) == fixture_bytes`. Byte-identity is the
conjunction (Aiken encoder ≡ generated bytes ≡ Haskell encoder), and the drift
check forbids the committed fixtures from diverging from a regenerate. This is the
only honest way to claim Aiken/Haskell parity — with executable evidence, not
prose.

  `aid_asset_name`/`spent_txid` context fields, the seven equalities (incl.

  `deriveAidAssetName` derivation, the Freshness "no `identity_root`/sliding window"
  statement, and the #81 no-`delegator` statement. STRICT in `gate.sh`.
- `threshold` — the Haskell threshold golden + all F18 negatives (incl. the exact
  `MAX_WEIGHT_DENOM` bound and unreduced-rational rejection) + positional-order
  sensitivity pass.
- `datum` / `messages` — the Haskell datum + message goldens and negatives pass,
  including stolen-current-quorum rejection, the cross-network/policy/asset and
  wrong-outref replay negatives, and the `deriveAidAssetName` golden +
  wrong-code/truncated/mutated-AID/substituted-asset negatives.
- `vectors` — regenerating vectors yields no `git diff` (fixtures in sync).
- `aiken` — `aiken check` passes and the Aiken threshold byte-identity tests pass.
- `parity` — Aiken datum/message `cbor.serialise` bytes equal the committed
  fixtures for every vector (cross-language byte-identity).
- `docs` — no surviving live `delegator`/`di` reservation or `identity_root`/
  sliding-window current-authority language in the reconciled surfaces; the
  Docs-links checker is GREEN.
- `final` — every target above. Tolerated RED in `gate.sh` until Slice 9 makes it
  strict.

## Risks / mitigations

- **Aiken/Haskell Plutus-`Data` encoding divergence** (empty-list, small-int,
  constructor-tag edge cases) → the drift + parity checks catch it at slice time;
  keep every field ≤32 bytes and records as `Constr` (no CBOR maps) to remove
  ordering ambiguity.
- **Doc-links CI regression** (em-dash→slug, moved anchors) → Slice 8 runs the
  full Lychee `--include-fragments` before hand-back; any `docs/` repair stays
  pair-owned (never ticket-owner edited).
- **Slice too large** (Slice 3 spans datum+messages) → split by the driver on the
  natural datum/message seam if RED→GREEN cannot be one clean commit.
- **Scope creep toward #24** → the pair briefs forbid validators, redeemers,
  tx-context, MPFS, genesis BLAKE3, and witness-receipt verification wiring.
